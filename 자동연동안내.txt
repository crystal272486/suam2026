수암초 GitHub Pages ↔ Notion 자동연동 설정

[자동연동 구조]
Notion 데이터 수정
→ GitHub Actions가 약 15분마다 Notion API 조회
→ data/class1.json 자동 갱신
→ GitHub Pages에 최신 작품 표시

==================================================
1. ZIP 파일 업로드
==================================================

1) 이 ZIP의 압축을 풉니다.
2) 내부 파일과 폴더를 suam2026 저장소 최상위에 모두 업로드합니다.
3) 숨김 폴더인 .github도 반드시 업로드해야 합니다.
4) 기존 파일은 덮어쓰기 합니다.

저장소 최상위 구조 예시:

.github/
  workflows/
    notion-sync.yml
scripts/
  sync-notion.mjs
data/
  class1.json
index.html
index.htm
class1.html
class1.htm

class2.htm

class3.htm
class4.html
class4.htm
.nojekyll

==================================================
2. Notion 토큰 만들기
==================================================

1) Notion Developer Portal을 엽니다.
2) 새 토큰 또는 새 내부 연결을 만듭니다.
3) 최소한 '콘텐츠 읽기(Read content)' 권한을 허용합니다.
4) 발급된 ntn_... 토큰을 복사합니다.
5) 토큰은 HTML이나 GitHub 일반 파일에 절대 입력하지 않습니다.

==================================================
3. 노션 데이터베이스를 연결에 공유
==================================================

1) 6학년 1반 원본 데이터베이스를 노션 편집 화면에서 엽니다.
2) 우측 위 ··· 메뉴를 누릅니다.
3) '연결 추가(Add connections)'를 누릅니다.
4) 2단계에서 만든 연결을 선택합니다.

공개 게시 설정과 API 연결 권한은 별개입니다.

==================================================
4. GitHub Secrets 등록
==================================================

suam2026 저장소에서:

Settings
→ Secrets and variables
→ Actions
→ Secrets
→ New repository secret

아래 2개를 만듭니다.

이름: NOTION_TOKEN
값: ntn_...으로 시작하는 실제 Notion 토큰

이름: NOTION_DATABASE_ID
값: 73db70c5165f835e9b8a01cb69d432f2

NOTION_DATA_SOURCE_ID는 자동 탐색 실패 때만 추가합니다.

==================================================
5. 최초 수동 실행
==================================================

1) 저장소 상단 Actions를 엽니다.
2) 왼쪽에서 Notion Sync를 선택합니다.
3) Run workflow를 누릅니다.
4) 다시 Run workflow를 눌러 실행합니다.
5) 초록 체크가 나오면 성공입니다.
6) data/class1.json 파일의 수정 시간이 바뀌었는지 확인합니다.
7) GitHub Pages 화면에서 Ctrl+F5를 누릅니다.

==================================================
6. 자동 갱신 주기
==================================================

기본 설정은 약 15분 간격입니다.

노션 수정 직후 바로 반영하려면:
Actions → Notion Sync → Run workflow

==================================================
7. 오류별 확인
==================================================

401 unauthorized
- NOTION_TOKEN 값이 잘못되었거나 토큰이 만료된 경우입니다.

403 forbidden
- 연결에 콘텐츠 읽기 권한이 없는 경우입니다.

404 object_not_found
- 원본 데이터베이스에서 '연결 추가'가 되지 않은 경우가 가장 많습니다.
- NOTION_DATABASE_ID가 정확한지 확인합니다.

git push denied
- 워크플로의 contents: write 권한이 차단된 경우입니다.
- Settings → Actions → General → Workflow permissions에서
  Read and write permissions를 선택합니다.

Run workflow 버튼이 안 보임
- .github/workflows/notion-sync.yml 파일이 main 브랜치에 올라갔는지 확인합니다.
